Create a new note with auto-generated frontmatter in the default note directory.

## When to use
- User asks "write a note about X" / "start a doc on Y".
- Committing a distilled idea, summary, or draft to long-term workspace memory.
- Capturing a decision or outcome from the current conversation.

## Do NOT use for
- **Overwriting an existing file** — this tool will fail if the path already exists. Use `edit_file` to modify, or `write_file` if you explicitly want to replace.
- **Plain text files without frontmatter**: use `write_file`.
- **Bulk import from external sources**: use `import_file` / `import_url` which run through the ingest pipeline (cleaner extraction, larger handling).

## Parameters
- `title` (string, optional, default `"Untitled Note"`): human-readable title; stored in frontmatter and used to derive the filename when `path` is omitted.
- `path` (string, optional): workspace-relative path (must end with `.md`). If omitted, the tool builds `{default_note_dir}/{slugify(title)}.md` (default note dir comes from user preferences, usually `Inbox`).
- `content` (string, optional): full body after frontmatter. Omitting it produces a skeleton body (`# {title}\n\nGenerated by more agent.`).

## Output
- `summary`: "Created a new note at <path> with title <title>".
- `citations`: the new note path.
- `payload.note`: full `NoteMeta` including auto-assigned `id`, `created_at`, `source_type="agent"`.
- Events: `note_created` (so the frontend sidebar updates live) + `tool_finished`.

## Chaining
- `create_note` → `update_note_metadata`: add tags / summary / related after creation.
- `create_note` → `link_notes`: connect the new note to existing ones.
- `create_note` → `read_note`: verify the file landed with expected content.

## Conventions
- `source_type` is fixed to `"agent"` so you can distinguish agent-authored notes from user-authored ones via `read_note`.
- Slug rules: lowercase, diacritics folded, non-alphanumeric (incl. CJK) collapsed to `-`. If the title contains only special characters, slug falls back to `"untitled-note"`.

## Errors
- Path collision: if the slugified path already exists, the tool errors. Either pass an explicit `path`, or check with `list_directory` first.
- Permission / FS errors: will bubble up as `ok=false`.
